Module Ardes::ActionView::InheritViews
In: lib/ardes/inherit_views.rb

Modifications to ActionView required by inherit_views

There is a new method render_parent, which renders first view that can be found higher in the inherited view paths with the same name as the current view being rendered 

  <%= render_parent %>
  <%= render_parent :locals => {:foo => foo} %>
  <%= render_parent :layout => false %>

To enable this functionality, there is a new instance variable _@current_render_ which holds the name of the file currently being rendered (similar to @first_render). In tempate ‘views/foo/bar.rhtml‘ 

  <%= @current_render %>

will produce ‘foo/bar.rhtml‘

Calls to render will function as normal within templates, as well as searching for inherited views when a template can’t be found
