rest_controller

ActionController extension to facilitate inheritance and DRYness in your rest controllers. REST style actions tend to follow a pattern of 

 * do the action
 * depending on whether the action succeeded or failed, respond accordingly

One problem is that, when sharing functionality via inheritance, an action has got too much stuff in it. You might want to simply offer an extra type of response - say an atom formatted repsonse to ‘show’. Normally you would have to rewrite the show method, duplicating a lot of code.

Simple Usage (see class docs for more)

class ThingsController < ApplicationController 

  rest_for :things

  respond_to :show do |format, status|
    format.atom { # add an atom feed to this controller's show action }
  end

  respond_to :show do |format.status|

end

Specs and Coverage

  • SPECDOC lists the specifications
  • RSpec Report is a report of spec pass/fails
  • RCov Report is the rcov report

Dependencies

  • RCov To install do: gem install rcov
  • RSpec To install do: gem install rspec
  • rspec rails plugin see rspec.rubyforge.org/documentation/rails/install.html

Testing

RSpec is used for testing, so the tests are in spec/ rather than test/ To run the tests you need a test database set up, edit spec/database.yml to provide the details.

Rake tasks

The following tasks are defined:

  • rspec (the default task) run the specs
  • rcov generate a coverage report (in doc/coverage)
  • rspec_rdoc generate an rdoc formatted spec doc (in SPECDOC)
  • rspec_rep generate a html report of passing/failing test cases (in doc/rspec_report.html)
  • rdoc generate the docs, (including SPECDOC)
  • rdoc_all generate all documentation including coverage and rspec report
